javaScript之变量,作用域,事件总结

javaScript

    基本类型和引用类型

    基本类型值

  • 基本类型值:简单的数据段
    • 按值访问,所以可以操作保存在变量中的实际的值.
    • 例如: undefined \ Null \ Number \ String
  • 传递基本类型的值时: 被传递的值会被赋值给一个局部变量

    引用类型值

  • 引用类型值: 多个值构成的对象
    • 引用类型的值是保存在内存中的对象,JS中是不能直接操作对象的内存空间,操作对象时,实际上是在操作对象的引用而不是实际的对象.所以,引用类型的值是按引用访问的
    • 例如:
      var person = new Object();
      person.name = "Nicholas";
  • 传递引用类型的值时,会把这个值在内存中的地址赋值给一个局部变量,这个局部变量的变化**

事件总结

事件流

  • 从页面中接收事件的顺序
  • IE的事件流是事件冒泡
  • Netscape的事件流是事件捕获

事件冒泡

  • 事件开始时由最具体的元素接收,然后逐级向上传播到较为不具体的节点

事件捕获

  • 不太具体的节点应该更早的接收到事件,二最具体的节点应该最后接收到事件.
  • 用意在于在事件犯法预定目标之前捕获它


DOM0级事件处理程序

  • 将一个函数赋值给一个事件处理程序属性
  • 例如: 为按钮绑定一个点击事件 btn.onclick = function(){ }
  • 其事件处理程序会在所属元素的作用域内运行


DOM2级事件处理程序

  • 定义了两个方法, addEventListener() 和removeEventListener()
  • addEventListener(事件名,事件函数,布尔值)
    • 用于处理指定程序的操作
    • 事件名: 要绑定的事件类型: click / mouseover / mouseout ………
    • 事件函数: 为其添加的事件函数名
    • 布尔值: 支持冒泡(false) / 支持捕获 (true)
    • [注意];1. 大多数情况下,都是讲事件处理程序添加到事件流的冒泡阶段,这样可以最大限度地兼容各种浏览器.
    • [注意]: 2. 通过这个方法添加的事件,只能通过removeEventListener 来移除事件
    • [注意]: 3. 通过这个方法添加的匿名函数将无法移出\
  • removeEventListener(事件名,事件函数,布尔值)
    • 用于删除指定程序的操作 所传参数必须与addEventListener参数相同
    • 事件名: 要绑定的事件类型: click / mouseover / mouseout ………
    • 事件函数: 为其添加的事件函数名
    • 布尔值: 支持冒泡(false) / 支持捕获 (true)


#IE事件处理程序

  • 实现了与DOM中类似的方法: attachEvent()和detachEvent()

  • attachEvent(事件名,事件函数)

    • 第一个事件名应该是onclick 注意与addEventListener()区分
    • 由于IE8及更早的版本只支持事件冒泡,所以通过attachEvent()添加的事件处理程序都会被添加到冒泡阶段
    • 事件处理程序会在全局作用域中运行.因此this = window
    • 对于同一个按钮多次添加attachEvent()方法,即添加多个事件处理程序,但是,是以相反的顺序被触发
    • 同样,匿名函数无法删除

  • detachEvent(事件名,事件函数)

    • 删除attachEvent添加的函数,必须提供相同的参数

补充: attachEvent() 与 DOM0级方法主要区别在与事件处理程序的作用域;


跨浏览器的事件处理程序

  • 要保证处理事件的代码能在大多数浏览器下一致地运行,只需要关注冒泡阶段
  • 创建的方法为addHander()和removeHander()
  • addHandler(要操作的元素,事件名称,事件处理程序函数) —– 添加事件
    • 主要的职责是视情况分别使用DOM0级方法,DOM2级方法或IE方法来添加事件
    • 该方法属于EventUtil对象,见书P354
  • removeHandler —– 删除事件
    • 同样接收相同参数
