HTML5

  • 历史记录,本地存储,拖拽,定位等简介

    自定义属性 data

  • h5获取元素上的所有的自定义属性dom.dataset
  • 获取具体的自定义属性dom.dataset . ( 自定义属性的名称 遵循驼峰命名法 )
  • 设置具体的自定义属性 dom.dataset . (自定义属性的名称 遵循驼峰命名法) = ‘ 值 ‘ ;
  • 都会去操作dom元素

    jquery中的data( ) 和自定义属性相关

    1. $(“ news”).data(“userName”); 获取自定义属性
    2. $(“news”).data(“userName”,”要修改的参数值”); 设置属性
  • 注意:

    jquery 修改自定义属性,是修改内存中的值(和变量类似),不会改变DOM元素节点的值;
    h5修改自定义属性dataset,是直接修改DOM中的元素值

历史记录 — history

  • history.pushState( null,null,"需要追加的记录")
  • history.replaceState( null, null , "需要替换的记录");

    注意: 以上操作均不会页面跳转

  • popstate 监听地址栏改变事件
  • window.onpopstate = function(){ }

本地存储—localStorage

  • 不会随着浏览器的关闭而消失,除非是主动清空数据
  • 仅在当前窗口有效
  • 有效范围: 同一网站的网页可以访问
  • localStorage-setItem(" 数据名", " 数据值"); 设置数据与存储
  • localStorage-getItem("数据名" ,"数据值") 获取数据
  • localStorage-removeItem(" 数据名"); 删除数据
  • localStorage-clear();清空数据
  • localStrage.length 返回对象数量
  • localStrage.key(i) 返回Key值

会话存储 — sessionStorage

  • 当浏览器关闭时就失效
  • 有效范围: 仅局限于当前页面
  • 不同浏览器窗口共享
  • 使用方法同上

cookie 与本地存储区别的问题

  1. cookie在浏览器和服务器之间来回传递,而sessionStorage和localStorage不会自动把数据发给服务器,仅在本地保存.
  2. cookie数据还有路径(path)的概念,可以限制cookie值属于某个路径下.存储大小显示也不同,cookie数据不能超过4K,同时因为http请求都会携带cookie,所以cookie只适合保存很小的数据,如会话标识.sessionStorage和localStorage虽然也有存储大小的限制,但比cookie大得多,可以达到5M或更大.
  3. 数据有效期不同,sessionStorage:仅在当前浏览器窗口关闭前有效 ; localStorage : 始终有效,窗口或浏览器关闭也一直保存,因此用作持久数据;cookie只在设置的cookie过期时间之前一直有效,即使窗口或浏览器关闭.
  4. 作用域不同,sessionStorage不在不同的浏览器窗口中共享,即使是同一个页面;localStorage在所有同源窗口中都是共享的;cookie也是在所有同源窗口中都是共享的.Web Storage支持事件通知机制,可以将数据更新的通知发送给监听者.Web Storage的API接口使用更方便.

cookie

  • cookie属于浏览器中的一个存储空间，里面可以存储数据，可存，可取
  • cookie是以域名为单位存储的，每个域名的cookie之间是相互隔离的，也就是说不可以互相操作
  • cookie是有容量限制的，每个域名可以存储的cookie不能超过4kB

  • 有时限，有生命周期。默认情况下cookie的生命会在浏览器关闭的时候结束。

    • 也可以手动的设置cookie的过期时间

  • cookie分路径的，子集路径是可以访问父级路径中存的cookie，但是父级路径不能访问子集路径中存储的cookie

    • 一般情况下，都会直接将cookie存储到根目录

  • cookie存储的路径是可以设置的

  • 当前域名中所有的cookie会伴随着每一次请求发送给服务器

session

  • session是服务器端的一个存储空间
  • 为每一个客户端划分了一些小空间，每一个小空间都会有一个sessionid
