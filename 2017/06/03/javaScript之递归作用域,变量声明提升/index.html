javaScript之递归作用域,变量声明提升

javaScript
  • 递归作用域 变量声明提升
  • 递归作用域 变量声明提升

    递归

  • 在函数中直接或者间接的调用给自己
  • 多用于解决数学问题 , 例如求前n 项和
  • 两大要素
    • 必须要有自己调用自己
    • 必须要有结束条件
  • 划归思想
    • 由难到易的过程,由复杂化简的过程成为划归

      作用域

  • 变量起作用的范围,就是变量的作用域
  • 在JS中有且只有函数可以创建作用域
  • 全局作用域: 直接写在script标签中的内容,就属于全局作用域
  • 局部作用域:写在函数中的内容,属于局部作用域的范围
  • 块级作用域: (js中没有块级作用域),通过代码块限定的作用域就是块级作用域,

词法

(静态作用域)

  • 是作用域的分类
  • 函数中变量的访问,只和函数声明的位置有关系,也就是和代码的声明书写结构有关系,和函数的具体调用,运行时的状态没有关系,这种作用域就是词法作用域
  • 和词法作用域相对的作用域叫做动态作用域

作用域链

  • 函数可以从创建作用域,函数中可以声明函数,这样就形成了作用域嵌套作用域的一个链式的访问结构,称作作用域链

变量的访问规则(搜索原则)

  • 当使用一个变量的时候,遵守如下的规则
    1. 现在使用变量的作用域中进行查找,如果找到了该变量的声明,就直接使用该变量
    2. 如果没有找到,就去上一级作用域中继续查找
    3. 如果没有找到,就继续沿着作用域链向外查找,知道找到了全局作用域位置

变量提升的特殊情况

  1. 函数同名的情况
    • 当在变量提升的时候,遇到函数同名的时候,后面的函数会把前面的函数给覆盖掉
  2. 函数和变量同名的情况
    • 当函数和变量同名的时候,只提升函数,忽略掉变量
  1. 变量是提升是分作用域的
  2. 变量提升是分段的(script标签)
    • 变量声明提升是会提升到当前的script标签内.
    • 注意: 两端函数时在同一作用域中的,可以相互访问.只是在预解析阶段是分开的

  3. 条件式函数声明

    • 在条件语句中声明的函数,就是条件式函数声明
    • 条件式函数声明在提升的时候,会被当做函数表达式来处理
    • 提升后的效果就是,只有函数名被提升了,函数体没有被提升

  4. 函数形参在变量提升过程当中的处理

    • 在做变量提升的时候,一次去找变量以及函数的声明,将其放在作用域的最顶上
    • 在进行变量提升之前,形参的声明以及赋值其实已经完成了.
    • 所以形参的声明和赋值不参与变量提升
